Current Truth

Facturel is a local Electron app for tracking bills and payment history with no cloud connectivity, no banking API, and no login. The codebase is a scaffold from December 2025 — it has the full intended shape (an App.js shell, a BillContext for state, a database.js SQLite manager, Electron main.js/preload.js, and five UI components: Dashboard, BillForm, BillDetails, PaymentForm, Sidebar) but none of it has been touched since the initial commit.

Every commit since December has been tooling-only: an Electron bump (27 → 35.7.5 via Dependabot), CI / Claude GitHub Actions workflows, README edits, and June checkpoint commits. No feature work has landed in src/ in over six months.

The clearest spec for what the app must do is the March 2026 diary, which documents the real bill-paying workflow using Chronicle Pro: open the app, review each active bill's last payment, click through to the payee site, pay it, then log date/amount/notes. That six-step loop is the bar facturel needs to clear — and currently does not.

What It Does (designed capability)

Local-only bill management: add payees with name, logo, due date, payment URL and notes; log payments with amount, date and notes; and keep an editable payment history. No cloud, no banking integration, no account — all data lives in a SQLite database in the user's app data directory. It supports recurring billing (monthly, quarterly, yearly) with automatic next-due tracking, a user-defined tagging system, bill archiving/reactivation, payment statistics (highest / lowest / average, year-over-year totals), and CSV export. The target user mirrors a Chronicle Pro user who wants the same visual simplicity but full local control. Ships cross-platform on Mac and Windows via Electron.

The Reference Workflow (from the March 2026 diary)

The bill-paying loop this app exists to replace — documented while paying Amex & CIBC bills in Chronicle Pro:

  1. Open app → see the list of active bills
  2. For each bill, review the last payment date and amount
  3. Click through to the payee website (payment URL)
  4. Pay the bill
  5. Log the payment: date + amount + notes
  6. Repeat for all bills
UX target: Chronicle Pro — visual simplicity and an at-a-glance active-items list. The PRD names the same "Chronicle" UI as its visual inspiration.
